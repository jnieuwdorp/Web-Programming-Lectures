htmlspecialchars()

  • Converts special characters < and > into &lt; and &gt;
  • What if a user does not provide the expected input?
  • http://www.st.ewi.tudelft.nl/~hauff/add2.php?first=<script>alert('Help!')</script>&second=4
    • code injection: <script>alert('Help!')</script> as one of the form's parameters
  • A malicious user can execute cross-site scripting attacks (XSS)
  • Use htmlspecialchars() to avoid such problems
http://www.sitepoint.com/php-security-cross-site-scripting-attacks-xss/

GET vs. POST

  • GET/POST create an associative array
    • first=>3, second=>4
  • key: name of form control, value: input data
  • in PHP accessed through $_GET and $_POST (superglobals)
  • $_GET passed via URL parameters
    • http://www.rottentomatoes.com/search/?search=life+of+pi
    • http://www.nu.nl/zoeken/?q=greenpeace
    • www.bing.com/images/search?q=amsterdam&qft=+filterui:color2-bw+filterui:aspect-square
  • $_POST passed to the script via HTTP POST method

GET

  • Parameters limited by the maximum URL length (~2000 characters)
  • Rule: do not pass sensitive data through GET - data is visible through URL!
  • Useful for non-secure data, e.g. Google/Bing/... search strings
  • Simplifies bookmarking for users (no need to refill the forms)
